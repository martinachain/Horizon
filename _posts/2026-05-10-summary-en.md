## [FreeBSD execve() Privilege Escalation Bug Due to Missing Parenthesis](https://www.freebsd.org/security/advisories/FreeBSD-SA-26:13.exec.asc) ⭐️ 8.0/10

A local privilege escalation vulnerability (CVE-2026-7270) was discovered in FreeBSD's execve() system call, caused by a missing parenthesis in a memmove() call that leads to incorrect memory copying. The bug allows an unprivileged user to execute arbitrary code with kernel privileges and gain root access. This vulnerability affects a widely-used operating system kernel, enabling local attackers to fully compromise the system. It underscores the importance of careful coding practices in security-critical code, especially operator precedence in C. The bug was introduced in a memmove() call where the size argument was computed as 'args->endp - args->begin_argv + consume' instead of 'args->endp - (args->begin_argv + consume)', causing an incorrect memory region to be copied. The vulnerability was patched in FreeBSD 15.0R-p7 and later updates.

hackernews · Deeg9rie9usi · May 9, 20:31 · [Discussion](https://news.ycombinator.com/item?id=48077971)

**Background**: The execve() system call is used to execute programs in Unix-like operating systems. It handles argument and environment strings, and a bug in its memory management can allow an attacker to overwrite kernel memory. Privilege escalation vulnerabilities are critical because they allow a regular user to gain full administrative control.

## [Let-go: A Clojure-like language in Go boots in 7ms](https://github.com/nooga/let-go) ⭐️ 8.0/10

Let-go is a new Clojure-like language implemented in pure Go that achieves a cold boot time of ~7ms, roughly 50x faster than JVM Clojure and 3x faster than Babashka. It is approximately 90% compatible with JVM Clojure and includes an nREPL server, making it usable with Clojure IDEs like Calva and CIDER. This project bridges the gap between Clojure's expressiveness and Go's performance and portability, offering a fast-starting, embeddable Lisp for CLIs, web servers, and scripting. It provides a viable alternative for Clojure developers who need native binaries and rapid startup without sacrificing the language's core features. Let-go ships as a ~10MB static binary and includes a custom compiler and stack VM, supporting AOT compilation to portable bytecode or standalone binaries. It can be embedded in Go programs with seamless interop for functions, structs, and channels, but does not load JARs or provide full Java API compatibility.

hackernews · marcingas · May 9, 17:52 · [Discussion](https://news.ycombinator.com/item?id=48076815)

**Background**: Clojure is a dynamic, functional Lisp dialect that runs on the Java Virtual Machine (JVM), known for its immutability and concurrency support. However, JVM startup time can be slow, motivating alternatives like Babashka (a native Clojure interpreter using GraalVM) and now Let-go, which uses Go's runtime for faster boot and smaller binaries. nREPL is a network REPL protocol widely used in the Clojure ecosystem for interactive development.

## [LLMs Corrupt Documents When Used for Delegation](https://arxiv.org/abs/2604.15597) ⭐️ 8.0/10

A new study introduces DELEGATE-52, a benchmark simulating long delegated workflows across 52 professional domains, and finds that each pass of an LLM degrades the intent and precision of documents, akin to repeated JPEG compression. This highlights a critical limitation of LLMs in automated document processing, undermining trust in delegation tasks and urging the design of agents that minimize round trips to the LLM. The study tested a basic agentic harness with file reading, writing, and code execution tools, but found that tool usage did not prevent corruption. The phenomenon is termed 'semantic ablation'—the removal of genuine, high-entropy signals from text.

hackernews · rbanffy · May 9, 08:44 · [Discussion](https://news.ycombinator.com/item?id=48073246)

**Background**: LLMs are often used in delegated workflows where they edit or rewrite documents. However, each pass tends to remove rare words, vivid metaphors, and precise details, a subtractive bias called semantic ablation. This is analogous to repeatedly saving a JPEG image, which degrades quality over time.
